# 最近范围筛选设计

## 目标

在网页仪表盘筛选栏新增“最近”范围能力。用户可以通过下拉快速选择 `1天`、`1周`、`1个月`、`3个月`、`半年`、`一年`，也可以在输入框手动填写 `<n>天`。

## 交互

- 保留现有 `今日`、`本周`、`本月`、`全部`、`自定义`。
- 新增一个 `最近` 控件，由文本输入框和下拉框组成。
- 选择下拉项或修改输入框时，将当前范围切换为 `recent`。
- 下拉项中 `1个月`、`3个月`、`半年`、`一年` 按自然跨度回退；`1天`、`1周` 按天数回退。
- 手动输入只支持形如 `<n>天` 的值，例如 `14天`。

## 数据流

- 前端状态新增 `recentValue`，服务模式下通过 `recentValue` 查询参数传给 `/api/usage`。
- 静态快照模式使用前端同一套范围语义过滤内嵌 report。
- 服务端 `usage-core` 接收 `preset=recent` 和 `recentValue`，解析后计算开始和结束时间。
- 月份回退需要夹到目标月份最后一天，避免月底日期溢出。

## 测试

- 服务端汇总覆盖自然月、半年和手填天数。
- 前端静态汇总覆盖 `recent` 模式，确保与服务端口径一致。
- 回归运行现有 `node --test`。
